1.Strategies applied

8 stages from the Session 4 Cleaning Pipeline Map were run, in order: Extract, Normalize, Language ID, Quality filter, Deduplicate, PII scrub, Decontaminate, Manifest.

#StrategyWhat it fixesWhat I actually found in this dataset
1ExtractPull the real content out of wrapper/packaging format.Only one text column exists; parsed every row into role turns (system/user/assistant) for the later stages to inspect. No wrapper noise to strip.
2NormalizeCanonicalize Unicode and strip broken characters.NFC-normalized all text. Found 1 row with raw control characters embedded mid-sentence and 1 row using fullwidth ＜think＞/＜/think＞ tags instead of ASCII — both fixed in place.
3Language IDCatch mislabeled or unexpected-language content.255 rows (3.3%) are <90% ASCII. Inspected a sample: these are legitimate translation-instruction tasks (Tamil, Thai, Russian, Chinese) and CJK punctuation, not corruption — tagged rather than dropped.
4Quality filterDrop malformed or degenerate samples.Checked <|im_start|>/<|im_end|> balance (0 mismatches), <think> tag balance (0 after normalize fix), empty assistant turns (0). Dropped 6 rows under 200 chars and 15 rows over 20,000 chars as length outliers.
5DeduplicateRemove repeated examples.6 exact duplicate rows found and removed (e.g. two identical "Kofi standing on a hillside" and "Nia in a swimming pool" prompt/response pairs). Whitespace-normalized re-check found no further near-duplicates.
6PII scrubMask emails, phones, and other personal data.43 rows matched an email-shaped pattern, 18 matched a phone-shaped pattern. Manually verified every hit: all are synthetic placeholders (@example.com, @test.com, @acme.com, your@email.com), version strings misparsed as emails (pnpm@9.1.0), or toll-free hotline numbers (e.g. 800-799-7233). Zero real PII found — nothing masked.
7DecontaminatePrevent public benchmark leakage.No offline eval-set fingerprint database available, so this was done heuristically: 3 rows match an NLI "Premise:/hypothesis" template (ANLI/RTE-style) and 8 rows contain an "Available choices:" multiple-choice template. Flagged for manual review, not auto-removed — could plausibly be repackaged GLUE/ANLI/SuperGLUE items.
8ManifestMake the run reproducible and auditable.Wrote manifest.json with per-stage counts, retention rate, final length distribution, system-prompt mix, and a SHA-256 content hash (2824407f6f3dfa3b…) of the cleaned corpus.

2.Dataset picked

Primary pick

lordx64/reasoning-distill-opus-4-7-max-sft

Downloaded as a single parquet shard (train-00000-of-00001.parquet, 15.8 MB, 7,823 rows, 1 column). Each row is a full ChatML conversation with an explicit reasoning trace in the assistant turn — a distillation-style SFT set built from a reasoning model's outputs.

Why it's a good fit for this exercise

It's a real, messy, downloaded dataset (not a toy CSV): mixed system-prompt schemas, mixed languages, occasional malformed unicode, synthetic PII-shaped strings, and benchmark-template-shaped questions — so every Session 4 stage had something genuine to check rather than a hypothetical.

Two system-prompt variants coexist in the raw data: "You are a helpful AI assistant." (2,121 rows) and "You are an assistant that responds to the User" (1,677 rows). The remaining 3,998 rows (51%) have no system turn at all — a valid third format for this corpus, not an error.

3.What was cleaned, why, and how

Cleanup areaWhy it mattersApplied action
Fullwidth reasoning tagsA stray row used ＜think＞/＜/think＞ (fullwidth Unicode) instead of ASCII <think>/</think>, which would silently break any tokenizer or parser expecting the literal ASCII tag.Normalized fullwidth variants to ASCII during the Normalize stage (1 row fixed).
Control charactersOne row embedded raw non-printable control bytes mid-sentence (mojibake-adjacent garbage from a bad source encoding), which would corrupt tokenization.Stripped C0 control characters except standard whitespace (1 row fixed).
Length outliersRows under 200 chars are too degenerate to teach a useful instruction/response pattern; rows over 20,000 chars risk being scraped/duplicated bulk text rather than a genuine single exchange.Dropped 6 too-short and 15 too-long rows (21 total, 0.27% of the corpus).
Exact duplicatesRepeated identical prompt/response pairs waste training signal and inflate apparent dataset size.Removed 6 exact duplicate rows via hash-based dedup.
PII-shaped stringsAny dataset with emails/phone numbers needs verification before shipping, even if most turn out to be synthetic.Extracted every match, manually classified all 43 email-shaped and 18 phone-shaped hits — confirmed 100% synthetic/placeholder, so no masking was applied (masking real placeholders would have made the data less useful for nothing).
Benchmark-style templatesNLI premise/hypothesis and multiple-choice "Available choices" templates are the exact shape of public eval sets (ANLI, RTE, GLUE-style tasks); if any are verbatim eval items, training on them would invalidate downstream benchmark scores.Flagged 11 rows for manual decontamination review; left in the corpus since no local eval fingerprint DB was available to confirm/deny overlap.

4.Other concerns checked

  • Sensitive-topic content, not toxicity: searched for profanity/sensitive terms; found legitimate crisis-hotline numbers (e.g. National DV Hotline 800-799-7233) and mature-topic conversations handled by the assistant responsibly (e.g. row 500 — an explicit user disclosure met with a supportive, non-judgmental reply). No gratuitous toxic content found; nothing removed on this basis.
  • Multilingual content is a feature, not noise: the 255 non-ASCII-heavy rows are genuine translation-instruction tasks (Tamil→English, Thai, Russian UI text, Chinese formatting instructions). A naive "keep ASCII only" filter would have wrongly deleted legitimate training signal — tagged instead of dropped.
  • Schema consistency: confirmed all rows use one canonical ChatML schema (no mixed JSON/plaintext formats), so no schema-unification pass was needed beyond the two system-prompt variants and the no-system-prompt case, all of which are valid.
  • License/provenance: the parquet file carries no embedded dataset card or license metadata locally — recorded in the manifest as an open item to confirm against the Hugging Face dataset page before any redistribution.
